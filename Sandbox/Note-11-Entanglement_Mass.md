# Note 11: Entanglement & Mass

**Subject:** The Origin of Inertia
**Reference:** The Architect's Codex - Physics of Karma

---

## 1. Contact as Data Infection
*   **Mechanism:** Physical contact is "Entanglement Swapping".
*   **Virus Analogy:** When Object C touches Object A (which is linked to B), C inherits the link to B.
*   **Result:** A complex web of "Shared Data Debt" (Butterfly Effect).

## 2. Mass as Data Resistance
*   **Definition:** Mass is not substance, but **Data Resistance** (Complexity).
*   **Cause:** The more entangled an object is (more dependencies), the more "Processing Power" is required to update its state (Move it).
*   **Inertia:** The "Lag" caused by dragging a file with millions of linked dependencies.
