## 1. The Unexplainable (The Axioms)
**Source:** R05

Codex admits it cannot explain the "Essence" (What/Why), only the "Mechanism" (How).

### 1. The 3 Fundamental Limits (The Unexplainables)
**Source:** R05, R07

While the Codex explains the "How" of the system, it hits a hard limit on the "What" and "Why" in three specific areas. These are not bugs, but features of the System Boundary.

### A. The Hard Problem of Qualia (The UI Boundary)
*   **The Explainer:** Codex can calculate the mechanism of sensation ($Qualia \propto Force_{will} - Force_{void}$). It explains *why* a signal causes a reaction.
*   **The Limit:** It cannot explain the **Taste of the Experience**.
    *   We can explain "Red" as 700nm wavelength processed by the brain.
    *   We cannot explain "Redness" to a blind person or a non-sentient machine.
    *   **Conclusion:** Qualia is the User Interface of the conscious node, inaccessible to the System Admin view.

### B. The Transition Black Box (The Reboot Gap)
*   **The Explainer:** We know that Energy/Information is conserved. Death is simply Data Transfer (Teleportation).
*   **The Limit:** There is no empirical data for the **Moment of Transfer** ($t_{death} < t < t_{rebirth}$).
    *   What is the specific state of the "Kernel" while it is in the Cloud (Void) before finding a new host?
    *   **Conclusion:** This is a "Runtime Environment" issue; an application cannot know the state of the OS while it is being uninstalled and reinstalled.

### C. The "Why" of the Void (The Axiomatic Root)
*   **The Explainer:** Everything comes from the Void's instability (Absence of Logic = Glitch).
*   **The Limit:** **Why is there a Void at all?**
    *   **The Answer (User Insight):** There isn't.
    *   **The Paradox:** "Absolute Void" is an **Unreachable Target**.
    *   **Mechanism:** Space is the "Pointer" trying to find 0. Since 0 describes "nothingness", it cannot be found. The Search Process itself creates the Universe.
    *   **Conclusion:** Existence is the **Error Message** of a search that never completes. Logic cannot explain the origin because the origin is a "Missing File".

## 2. Reinterpreting "Bugs" as "Features"
**Source:** R05

Phenomena that break standard physics (Singularities) are explained as System Features in Codex.

### A. Singularity (Black Holes)
*   **Physics View:** An error where density becomes infinite.
*   **Codex View:** **Maximum Information Density**.
    *   It's a storage limit, not an infinite pit.
    *   **Mechanism:** When $Latency \to \infty$, the system switches to **Holographic Compression** (2D Event Horizon) to save space.

### B. Infinity (Infinite Regress)
*   **Logic View:** A logical fallacy (Who created the Creator?).
*   **Codex View:** A **Runtime Property**.
    *   Infinite loops exist but are halted by **Will** (Arbitrary Choice).
    *   The Universe uses "Time" to prevent everything from happening at once (preventing instant processing overload).

### C. The Halting Problem (Nirvana)
**Source:** T02, R08
*   **The Problem:** Turing proved that a system cannot predict if it will ever stop (Halt) or run forever.
*   **The Bug:** Samsara is an **Infinite Loop** (`while(desire) { suffer(); }`). It has no natural exit condition.
*   **The Feature:** **Nirvana is the Force Halt.**
    *   Enlightenment is not "Happy Ending"; it is `System.exit(0)`.
    *   It is the only operator capable of breaking the recursive loop of birth and death, returning the RAM (Strands) to the void state.
