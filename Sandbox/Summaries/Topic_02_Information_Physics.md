## 3. Gravity as Latency
**Source:** Note 8, 14, 20

*   **The Concept:** Gravity is not a "pulling force" but a manifestation of **System Lag** (Latency).
*   **Mechanism:**
    *   Mass = Data Density ($D$).
    *   High Mass = High Data Density = High Processing Load.
    *   When the system processes a heavy load (e.g., a star), it slows down. This "slowdown" is perceived as **Time Dilation**.
*   **The Slide:** Objects aren't pulled; they "slide" into the area of slower processing (higher latency) simply because the update rate there is lower.
*   **Equation:** $T_{obs} = T_{sys} \times \sqrt{1 - \frac{L_{load}}{L_{max}}}$
    *   Where $L_{load}$ is the processing load (mass). As load approaches max, time ($T_{obs}$) approaches 0 (Event Horizon).
