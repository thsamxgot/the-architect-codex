## 2. ของไหล (Fluids) คือ Adaptive Data Topology

**นิยาม:** สสารที่เปลี่ยนรูปร่างตามภาชนะบรรจุ
**Codex Definition:** Non-Rigid Data Structures

*   ของแข็ง (Solids) เปรียบเสมือน **Hard-coded Structures** (แก้ไขยาก, คงรูป)
*   ของไหล (Fluids) เปรียบเสมือน **Dynamic Arrays / Linked Lists** ที่ปรับเปลี่ยนโครงสร้าง (Topology) ได้ตาม Boundary Conditions
*   **Pascal's Principle:** แรงดันที่กระทำต่อของไหลในระบบปิด จะส่งผ่านไปยังทุกจุดเท่าๆ กัน
    *   นี่คือคุณสมบัติของ **Global State Update:** การเปลี่ยนแปลงค่า Parameter หนึ่งตัวในระบบที่เชื่อมโยงกัน จะส่งผลกระทบ (Propagate) ไปยังทุก Node ใน Network ทันที

## 1. ของไหล (Fluids) คือ Adaptive Data Topology

ใน **Chapter 4: The Stage Called Matter** เราอธิบายว่าสสารคือ "เชือกที่ขมวดปม" (Knots)
แล้ว "น้ำ" หรือ "อากาศ" ล่ะ? มันคือเชือกแบบไหน?

ของไหลในมุมมองของ Codex คือ **"เครือข่ายที่พันกันหลวมๆ" (Loosely Coupled Network)**
ต่างจากของแข็งที่เป็น "ปมตาย" (Fixed Knots) ที่ขยับยาก... ของไหลคือกลุ่มข้อมูลที่สามารถ **"Re-topology"** (เปลี่ยนรูปทรง) ได้ตลอดเวลาตามภาชนะที่ใส่

*   **ความหนืด (Viscosity):** คือค่า **Network Latency** ในการส่งผ่านข้อมูลระหว่าง Node เพื่อนบ้าน
    *   น้ำผึ้ง (High Viscosity) = High Latency (ข้อมูลส่งถึงกันช้า, เปลี่ยนรูปร่างยาก)
    *   น้ำเปล่า (Low Viscosity) = Low Latency (ข้อมูล Flow ลื่นไหล)

## 2. คลื่น (Waves) คือ Signal Propagation

คลื่นไม่ใช่ "วัตถุ" แต่เป็น **"อาการ"** ของการส่งต่อพลังงาน
เหมือนใน **Chapter 6: Karma Is Echo**
เมื่อคุณโยนหินลงน้ำ คุณกำลังส่ง "Action" ลงไปในระนาบของ **"The Stage"** (ผิวน้ำ)
แรงกระเพื่อมที่เห็น คือการที่ Node หนึ่งส่งค่า State Update ไปให้ Node ถัดไป (Chain Reaction)

*   **Wave Speed:** ขึ้นอยู่กับค่า **Update Rate** ของตัวกลาง (Bandwidth ของเส้นเชือกบริเวณนั้น)
*   **Interference (การแทรกสอด):** คือการบวก/ลบกันของข้อมูล (Data Superposition)
    *   คลื่นเสริม = ข้อมูลเสริมแรงกัน (Amplification)
    *   คลื่นหักล้าง = ข้อมูลขัดแย้งกันจนเป็น Null (Cancelation)

## 3. แรงลอยตัว (Buoyancy) คือ Density Sorting

ทำไมเรือเหล็กถึงลอยน้ำ? และทำไมลูกโป่งถึงลอยขึ้นฟ้า?
อาร์คิมิดีสบอกว่า "แรงลอยตัวเท่ากับน้ำหนักของของเหลวที่ถูกแทนที่"
แต่ใน Codex เรามองว่ามันคืออัลกอริทึม **"Density Sorting"** (การจัดเรียงข้อมูลตามความหนาแน่น)

*   จักรวาลพยายามจัดระเบียบตัวเองให้ "ข้อมูลที่หนัก" (High Density/Complexity) จมลงสู่จุดศูนย์ถ่วง (Bottom)
*   "ข้อมูลที่เบา" (Low Density) จะถูกดันขึ้นไปอยู่ข้างบน (Top)
*   แรงลอยตัวจึงไม่ใช่แรงผลัก... แต่เป็น **"แรงบีบ" (Pressure Gradient)** ของระบบที่พยายามดันสิ่งที่เบากว่าให้ "หลีกทาง" (Give way) แก่สิ่งที่หนักกว่า

## 4. หลักการของ Bernoulli (Bernoulli's Principle)

**หลักการ:** "ที่ไหนความเร็วสูง ความดันจะต่ำ"
**Codex Definition:** Bandwidth Allocation

*   เมื่อข้อมูลวิ่งเร็ว (High Velocity) ระบบต้องทุ่มทรัพยากรไปกับการ "ส่งข้อมูล" (Routing)
*   ทำให้เหลือทรัพยากรสำหรับ "การดันผนัง" (Static Pressure) น้อยลง
*   เหมือนท่ออินเทอร์เน็ตที่เต็มไปด้วย Traffic ขาออก... Traffic ขาเข้าหรือด้านข้างก็จะเบาบางลง
