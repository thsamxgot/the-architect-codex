# 03. Decoupling Sensation: The Override Mechanism
## (การแยกสัญญาณผัสสะ: กลไกการ Override ระบบ)

**Phase 3: The Player Interface (อินเทอร์เฟซผู้เล่น)**

> *"เจตจำนงสามารถสั่ง 'แยก' ความเจ็บ (Physical) ออกจากความทุกข์ (Mental) ได้เหมือน Admin Override"*

---

### The Code (รหัสพันธุกรรมแห่งความคิด)
Pain (ความเจ็บปวด) คือข้อมูลดิบ (Raw Data) ที่ส่งมาจาก Hardware เพื่อแจ้งเตือนความเสียหาย
Suffering (ความทุกข์) คือการตีความข้อมูลนั้น (Interpretation) ว่า "ฉันแย่แล้ว" "ฉันไม่ชอบมัน"

สมการคือ: **Suffering = Pain x Resistance**
(ความทุกข์ = ความเจ็บ x แรงต้าน)

ถ้า Resistance = 0, ความทุกข์ก็จะเป็น 0 แม้ความเจ็บจะยังมีอยู่ (Pain without Suffering)

### Decoding (ถอดรหัสเพื่อการนำไปใช้)

#### 1. The Second Arrow (ศรดอกที่สอง)
พระพุทธเจ้าเปรียบเทียบว่า:
- ดอกที่ 1: กายเจ็บ (Pain) -> หลบไม่ได้ เป็นระบบเตือนภัยธรรมชาติ
- ดอกที่ 2: ใจทุกข์ (Suffering) -> "ทำไมต้องเป็นเรา" "เมื่อไหร่จะหาย" -> นี่คือ Software Error ที่เราสร้างเอง

#### 2. The Separation Protocol (โปรโตคอลการแยกส่วน)
Architect ที่ฝึกฝนมาดี สามารถใช้คำสั่ง **Override** เพื่อแยก Pain ออกจาก Self
- มองความเจ็บเป็น "ก้อนพลังงาน" (Cluster of Sensation)
- อย่าเรียกว่า "ฉันเจ็บ" ให้เรียกว่า "มีความเจ็บปวดเกิดขึ้นที่ขา" (There is pain in the leg)
- การเปลี่ยน Subject ของประโยค คือการเปลี่ยนโครงสร้างสมองในการรับรู้

#### 3. Practical Application: "Pain Scanning"
- เมื่อปวดหัว หรือไม่สบายใจ อย่าพยายาม "ไล่" มัน (เพราะนั่นคือ Resistance)
- ให้ **"Zoom In"** เข้าไปดูมันชัดๆ ดูว่ามันหน้าตาเป็นยังไง มันหมุนยังไง มันมีสีอะไร
- เมื่อคุณจ้องมันด้วยความอยากรู้อยากเห็น (Curiosity) ความกลัวจะหายไป และความเจ็บจะกลายเป็นแค่ "ปรากฏการณ์ที่น่าสนใจ" (Interesting Phenomenon)
