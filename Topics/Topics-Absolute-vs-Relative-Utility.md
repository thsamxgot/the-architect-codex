# 08. Absolute vs. Relative Utility
## (อรรถประโยชน์สัมบูรณ์ vs สัมพัทธ์: กับดักของการเปรียบเทียบ)

**Phase 6: Optimization & Debugging (การปรับแต่งระบบ)**

> *"เงิน 1 ล้านบาท มีค่าไม่เท่ากัน ขึ้นอยู่กับว่าเพื่อนข้างบ้านมี 10 ล้าน หรือ 1 แสน"*

---

### The Code (รหัสพันธุกรรมแห่งความคิด)
ในทางเศรษฐศาสตร์ เราถูกสอนให้หา **Utility Maximization** (ความพอใจสูงสุด)
แต่สมองมนุษย์ถูก Hardcode มาให้มองแบบ **Relative** (เปรียบเทียบ) ไม่ใช่ **Absolute** (สัมบูรณ์)

- **Absolute Utility:** ฉันมีรถขับ = ฉันสบาย
- **Relative Utility:** ฉันมี Benz แต่เพื่อนขับ Ferrari = ฉันทุกข์

นี่คือ Bug ใน Code ของมนุษย์ที่ทำให้ "ยิ่งรวยยิ่งทุกข์" (Hedonic Treadmill)

### Decoding (ถอดรหัสเพื่อการนำไปใช้)

#### 1. The Benchmark Glitch (ความผิดพลาดของตัวเปรียบเทียบ)
เราชอบ Set Benchmark ผิดที่
- ตัวเปรียบเทียบ Default ของเรามักจะเป็น "คนที่เหนือกว่า" (Upward Comparison) -> ทำให้เรารู้สึกขาด
- ลอง Hack Code โดยการเปลี่ยน Benchmark เป็น "ตัวเราเมื่อวาน" หรือ "คนที่ลำบากกว่า" (Downward Comparison) -> Utility ของสิ่งที่มีอยู่จะพุ่งสูงขึ้นทันที

#### 2. Absolute Goal Setting (การตั้งเป้าหมายแบบสัมบูรณ์)
สร้างเป้าหมายที่ **Self-Contained** (จบในตัว) ไม่ขึ้นกับคนอื่น
- ผิด: "ฉันต้องรวยกว่าเพื่อนรุ่นเดียวกัน" (External Dependency)
- ถูก: "ฉันต้องมีอิสรภาพทางการเงินที่เกษียณได้" (Internal Metric)

#### 3. Practical Application: "Disconnect the Comparator"
- เมื่อรู้สึกอิจฉาใคร ให้รู้ทันว่า **Comparator Module** กำลังทำงานผิดพลาด
- บอกตัวเองว่า "เกมของเขา กับเกมของฉัน คนละกติกา (Different Ruleset)"
- เขาอาจจะเล่นเกมสะสมเงิน แต่คุณเล่นเกมสะสมความสงบ -> เอาแต้มเขามาเทียบกับแต้มเราไม่ได้ (Type Mismatch)
